“所有用户的输入都是不可信的”

这个思想指导我们在设计表单的时候，一定要进行用户输入的验证。对于用户体验而言，越早的反馈则越佳，所以表单验证的工作应该尽可能地在前端就进行（当然，前端对于后端而言也是输入端，所以后端仍然需要进行检验）。

表单验证器原理

每个输入值可以对应一个或者多个验证规则

$.validate(value, rules);
// value 用户的输入值   
//       例：'123456'
// rules 用户的输入值对应验证规则，可针对一个输入值多个验证规则
//       例：rules: ['#', '请输入5到10个值', '#rangelength', 5, 10]
//       以 #开头的为验证规则名称   后面跟着规则的内容  
//       例： '#rangelength', 5, 10 输入值5到10位数字

值规范验证

根据单个输入值进行验证数据合法性：要求一个输入框，输入不能为空并且输入值为5到10位数的数字

$.validate(
	'12345',
	['#', '请输入5到10位数字', '#rangelength', 5, 10]
);
// '12345' 需验证值内容   
//       需验证值内容
// ['#', '请输入5到10位数字', '#rangelength', 5, 10] 
//       验证值是否规范的规则

基本验证例子

例：根据单个输入值进行验证数据合法性：要求一个输入框，输入不能为空并且输入值为5到10位数的数字

var result = $('#subForm').name("number").validate(
	['#', '请输入5到10位数字', '#rangelength', 5, 10]
);

表单验证例子

例：根据单个输入值进行验证数据合法性：要求一个输入框，输入不能为空并且输入值为5到10位数的数字

$('#subForm').dataform({
	url: 'http://www.banbang.cn/add.jhtml',
	rules: {
		number: ['#', '请输入5到15位数字', '#rangelength', 5, 18],
		name: ['#', '您输入的内容不合法', '#rangelength', 8, 10]
	},
	proxy: function(url, data) {
		alert('您请求的地址为：'+url+'您发送的数据为：'+data);
	}
});

表单提交参数说明

序号 参数 说明 是否必须
1 proxy 数据验证成功后执行的方法，参数 url: 表单提交数据的url地址; data: 表单提交的数据内容 是
2 url 表单提交数据的url地址，如果url为空，自动获取form表单的action地址 否
3 data 表单提交的数据内容，如果表单的data为空，自动获取form系列化serialize数据 否
4 rules 表单数据的规则，例子： 
rules: {
	id: ['#', '请输入5到15位数字', '#rangelength', 5, 18],
	name: ['#', '您输入的内容不合法', '#rangelength', 8, 10]
}
否
5 error 数据验证后提示操作,一般默认为空，参数 el: 不规范数据元素，msg: 不规范数据提示消息 否
6 tip 定义不规范数据提示消息，一般默认样式，参数 el: 不规范数据元素，msg: 不规范数据提示消息 否

}

rules: {
	[value]: ['#regex', '{regex}', '{tip}']
	//  {regex}: 需要验证的正则表达式
	//  {tip}: 验证提示语
}
